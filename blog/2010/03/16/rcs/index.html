RCS - 昭和のバージョン管理システム

2010 年 3 月 16 日

RCS は 1980 年代初頭、 Walter F. Tichy によって発表 された。 ピンクレディーが解散を発表したのも、ちょうどその頃だ。

RCS を使うには、まず管理フォルダを作ってファイルをチェックインする。

% mkdir RCS
% ci hello.c
RCS/hello.c,v  <--  hello.c
enter description, terminated with single '.' or end of file:
NOTE: This is NOT the log message!
>> My great hello world program
>> .
initial revision: 1.1
done

簡単だね。

ところで、 この ハロー・ワールド・プログラムは素晴らしくクール なんだ。 これをライブラリにしない手はないだろう。 それにはヘッダファイルが必要だし、当然 Makefile も欲しい。

% ci hello.h
RCS/hello.h,v  <--  hello.h
enter description, terminated with single '.' or end of file:
NOTE: This is NOT the log message!
>> Header for hello world library
>> .
initial revision: 1.1
done
%
% ci Makefile
RCS/Makefile,v  <--  Makefile
enter description, terminated with single '.' or end of file:
NOTE: This is NOT the log message!
>> rules to build hello world
>> .
initial revision: 1.1
done

しばらくして、ハロー・ワールドに新機能を取り込むことになった。

% co hello.c
% rcs -l hello.c
RCS file: RCS/hello.c,v
1.1 locked
done
%
% chmod +w hello.c
% vi hello.c
... hack, hack ...
% ci hello.c
RCS/hello.c,v  <--  hello.c
new revision: 1.2; previous revision: 1.1
enter log message, terminated with single '.' or end of file:
>> the new hello world function
>> .
done

見て分かるように、 RCS のリビジョン番号はファイルごとにてんでバラバラだ。 RCS はファイルの関連を管理してくれない。これは、ファイルが 2 つ以上あるプロジェクトでは問題になる。
